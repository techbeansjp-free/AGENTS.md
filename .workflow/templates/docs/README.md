# システム仕様書

## ドキュメント構成

本システム仕様書は、以下のセクションで構成されています。

1. **[システム概要](./01_システム概要/README.md)**
   - システムの全体像、構成、業務フローを定義
2. **[画面設計](./02_画面設計/README.md)**
   - 画面の動きと、各画面の項目を定義
3. **[データ設計](./03_データ設計/README.md)**
   - ER図とテーブル定義を定義
4. **[機能設計](./04_機能設計/README.md)**
   - 複雑なロジックやバッチ処理を定義

---

## ドキュメントの読み方

### 対象読者

- **開発者**: 実装の詳細を理解するため
- **プロジェクトマネージャー**: 進捗管理と品質確認のため
- **クライアント**: システムの機能と仕様を理解するため

### 記述ルール

- **Mermaid 図**: システム構成、フロー、ER図などを視覚的に表現
- **表**: 画面項目、テーブル定義などを構造化して記述
- **Markdown**: テキストベースで記述し、Git でバージョン管理
